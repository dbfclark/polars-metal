# Hand-rolled MSL FFT Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A from-scratch Metal (MSL) 1-D FFT kernel — correct and fast for **all sizes** — that backs `.metal.fft()`/`.metal.ifft()`, removing MLX's 2^20 ceiling (ml-explore/mlx#1800).

**Architecture:** Port MLX's proven MSL FFT codelets into our own `shaders/fft.metal` (+ `_fft_radix.metal` header), driven by a Rust **planner+dispatcher** in `polars-metal-kernels` (mirrors MLX's `fft.cpp`). Build correctness-first and incrementally: radix base → mixed-radix → four-step (large pow2) → recursive four-step → Bluestein → Rader → engine integration. Every step is gated by a numpy differential test. **Never patch `vendor/mlx`** — its source is the reference, not the runtime.

**Tech Stack:** MSL, Rust (`polars-metal-kernels` dispatch via `ShaderLibrary`/`CommandQueue`), PyO3, numpy.

**Spec:** `docs/superpowers/specs/2026-06-09-m6-a3-msl-fft-design.md`.

**MLX reference (read the matching piece before each port; DO NOT copy verbatim — adapt to our buffer/dispatch conventions):**
- `vendor/mlx/mlx/backend/metal/kernels/fft/radix.h` — `complex_mul`, `get_twiddle`, radix_2..8 codelets.
- `vendor/mlx/mlx/backend/metal/kernels/fft/readwrite.h` — load/store + real-packing.
- `vendor/mlx/mlx/backend/metal/kernels/fft.metal` — entry points / function-constant specialization.
- `vendor/mlx/mlx/backend/metal/fft.cpp` — host planner (factorization, four-step, Rader, Bluestein).

## Conventions used throughout (read once)

**Kernel dispatch (from `rolling.rs`):**
```rust
use crate::command::CommandQueue;
use crate::shader_lib::shared_library;
use polars_metal_buffer::{MetalBuffer, MetalDevice};

let lib = shared_library(device)?;
let pso = lib.pipeline("fft_entry_point_name")?;
let n_buf = device.new_buffer_from_bytes(&(n as u32).to_le_bytes())?;   // scalar → constant uint&
let mut queue = CommandQueue::new(device)?;
queue.dispatch_1d_with_tg(&pso, &[input, output, &n_buf], n_threads, tg_width)?;
queue.wait_until_complete()?;
```
- Data buffers: `MetalBuffer::from_f32_slice(device, &slice)`; outputs: `device.new_buffer_zeroed(n_bytes)`.
- Scalars: 1-element little-endian `MetalBuffer` via `new_buffer_from_bytes`, bound as `constant uint& x [[buffer(k)]]`.
- Complex data is stored **interleaved** `float2` (re, im) — N complex points = `2N` f32.
- `dispatch_1d_with_tg` uses `dispatchThreads:` (auto-pads the trailing threadgroup); when a kernel needs full threadgroups, pad `n_threads` up to a multiple of `tg_width` and guard surplus threads in the kernel.
- Threadgroup width: pass the desired width; `dispatch_1d_with_tg` clamps to `pso.maxTotalThreadsPerThreadgroup()` at runtime (do NOT hardcode a device-specific max).
- MSL shaders auto-compile from `shaders/*.metal` via `crates/polars-metal-kernels/build.rs`; `_`-prefixed files are `#include`-only headers.

**Differential test harness (Rust, `tests/test_fft.rs`):** compute the FFT on the GPU, compare to a numpy-equivalent reference computed **in Rust** via a known-correct DFT for tiny sizes, and against precomputed numpy values for larger sizes. Tolerance = L2-relative-error ≤ `1e-4` (F32). For the Python-level sweep (`tests/python_integration/test_fft.py`) compare directly to `numpy.fft`.

All `cargo test` runs use `--test-threads=1` (Metal command-queue contention).

// shaders/_fft_radix.metal — FFT codelets (adapted from vendor/mlx/.../fft/radix.h).
// Header-only ('_' prefix): #included by fft.metal, not compiled standalone.
#pragma once
#include <metal_stdlib>
using namespace metal;

// Complex multiply: (a.x+i a.y)(b.x+i b.y).
inline float2 cmul(float2 a, float2 b) {
    return float2(a.x * b.x - a.y * b.y, a.x * b.y + a.y * b.x);
}
// Twiddle e^{-2πi k / p} (forward). For inverse, pass +sign via `inv`.
inline float2 twiddle(int k, int p, bool inv) {
    float theta = (inv ? 2.0f : -2.0f) * float(k) * M_PI_F / float(p);
    return float2(fast::cos(theta), fast::sin(theta));
}
```

//! M6 A3: hand-rolled MSL FFT — planner + dispatcher. See
//! docs/superpowers/specs/2026-06-09-m6-a3-msl-fft-design.md.
use crate::command::{CommandQueue, DispatchError};
use crate::shader_lib::{shared_library, ShaderError};
use polars_metal_buffer::{BufferError, MetalBuffer, MetalDevice};

#[derive(Debug, thiserror::Error)]
pub enum FftError {
    #[error("shader library: {0}")]
    Shader(#[from] ShaderError),
    #[error("dispatch: {0}")]
    Dispatch(#[from] DispatchError),
    #[error("buffer: {0}")]
    Buffer(#[from] BufferError),
    #[error("unsupported fft size {0}")]
    Unsupported(i64),
}

/// Naive O(N^2) DFT over interleaved-complex input `[re0,im0,re1,im1,...]`.
/// Reference oracle for tests ONLY (never a runtime path). `inverse` applies
/// +sign twiddles and 1/N scaling.
#[cfg(test)]
pub fn dft_reference(input: &[f32], n: usize, inverse: bool) -> Vec<f32> {
    let mut out = vec![0f32; 2 * n];
    let sign = if inverse { 1.0f64 } else { -1.0f64 };
    for k in 0..n {
        let (mut sre, mut sim) = (0f64, 0f64);
        for t in 0..n {
            let ang = sign * 2.0 * std::f64::consts::PI * (k as f64) * (t as f64) / (n as f64);
            let (c, s) = (ang.cos(), ang.sin());
            let (re, im) = (input[2 * t] as f64, input[2 * t + 1] as f64);
            sre += re * c - im * s;
            sim += re * s + im * c;
        }
        let scale = if inverse { 1.0 / n as f64 } else { 1.0 };
        out[2 * k] = (sre * scale) as f32;
        out[2 * k + 1] = (sim * scale) as f32;
    }
    out
}

/// L2 relative error between two interleaved-complex vectors (test helper).
#[cfg(test)]
pub fn l2_rel_err(got: &[f32], exp: &[f32]) -> f64 {
    let (mut num, mut den) = (0f64, 0f64);
    for i in 0..exp.len() {
        let d = got[i] as f64 - exp[i] as f64;
        num += d * d;
        den += (exp[i] as f64) * (exp[i] as f64);
    }
    (num / den.max(1e-300)).sqrt()
}
```

- [ ] **Step 3: Write the radix-2 Stockham kernel** in `shaders/fft.metal`:
```cpp
// shaders/fft.metal — hand-rolled FFT kernels (M6 A3).
#include "_fft_radix.metal"
using namespace metal;

// 1024 complex points × 8 B × 2 ping-pong buffers = 16 KB threadgroup memory,
// safely under Apple Silicon's 32 KB limit. (4096 would need 64 KB — too big.)
constant uint FFT_BASE_MAX = 1024;

// One threadgroup transforms one length-n pow2 signal (n <= FFT_BASE_MAX),
// iterative Stockham radix-2. Input/output interleaved float2 in global mem.
// `n` and `inv` (0/1) are scalar buffers. tg memory holds 2 working buffers.
kernel void fft_stockham_pow2_f32(
    device const float2* in   [[buffer(0)]],
    device float2*       out  [[buffer(1)]],
    constant uint&       n    [[buffer(2)]],
    constant uint&       inv  [[buffer(3)]],
    uint tid                  [[thread_position_in_threadgroup]],
    uint tg_size              [[threads_per_threadgroup]]) {
    threadgroup float2 a[FFT_BASE_MAX];
    threadgroup float2 b[FFT_BASE_MAX];
    // cooperative load
    for (uint i = tid; i < n; i += tg_size) a[i] = in[i];
    threadgroup_barrier(mem_flags::mem_threadgroup);
    threadgroup float2* src = a;
    threadgroup float2* dst = b;
    bool inverse = inv != 0;
    for (uint len = 2; len <= n; len <<= 1) {
        uint half = len >> 1;
        for (uint i = tid; i < n / 2; i += tg_size) {
            uint base = (i / half) * len;
            uint j = i % half;
            float2 u = src[base + j];
            float2 v = cmul(src[base + half + j], twiddle(int(j), int(len), inverse));
            dst[base + j] = u + v;
            dst[base + half + j] = u - v;
        }
        threadgroup_barrier(mem_flags::mem_threadgroup);
        threadgroup float2* t = src; src = dst; dst = t;
    }
    float scale = inverse ? (1.0f / float(n)) : 1.0f;
    for (uint i = tid; i < n; i += tg_size) out[i] = src[i] * scale;
}
```
> Note: this is the textbook Stockham layout; cross-check the butterfly indexing against `vendor/mlx/.../radix.h`'s `radix2` and the Stockham stage loop in `fft.metal`. The differential test is the arbiter.

- [ ] **Step 4: Implement `fft_gpu` (base path only)** in `crates/polars-metal-kernels/src/fft.rs`:
```rust
pub const FFT_BASE_MAX: i64 = 1024; // matches shaders/fft.metal (16 KB tg mem)

/// Compute a 1-D FFT over an interleaved-complex host slice `input`
/// (`[re,im,...]`, length 2*n). Returns the transform interleaved, length 2*n.
/// `inverse` applies 1/N scaling. (Phase 0: pow2 n <= FFT_BASE_MAX only.)
pub fn fft_gpu(
    device: &MetalDevice,
    input: &[f32],
    n: i64,
    inverse: bool,
) -> Result<Vec<f32>, FftError> {
    debug_assert_eq!(input.len() as i64, 2 * n);
    if n <= 0 || (n & (n - 1)) != 0 || n > FFT_BASE_MAX {
        return Err(FftError::Unsupported(n));
    }
    let in_buf = MetalBuffer::from_f32_slice(device, input)?;
    let out_buf = device.new_buffer_zeroed((2 * n as usize) * std::mem::size_of::<f32>())?;
    let n_buf = device.new_buffer_from_bytes(&(n as u32).to_le_bytes())?;
    let inv_buf = device.new_buffer_from_bytes(&u32::from(inverse).to_le_bytes())?;

    let lib = shared_library(device)?;
    let pso = lib.pipeline("fft_stockham_pow2_f32")?;
    let mut queue = CommandQueue::new(device)?;
    // One threadgroup; width = min(n, base). The kernel strides by tg_size.
    let tg = (n as usize).min(1024);
    queue.dispatch_1d_with_tg(&pso, &[&in_buf, &out_buf, &n_buf, &inv_buf], tg, tg)?;
    queue.wait_until_complete()?;
    Ok(out_buf.to_f32_vec())  // see note
}
```
> `out_buf.to_f32_vec()` — use the buffer crate's existing host-readback (find the real method on `MetalBuffer`; e.g. `as_f32_slice().to_vec()` or the same readback `rolling`/tests use). If none exists, add a small `to_f32_vec(&self) -> Vec<f32>` mirroring how tests read rolling output. The single-threadgroup dispatch (`n_threads == tg`) runs exactly one threadgroup.

## Phase 2 — Four-step for large power-of-2 (THE HEADLINE: fixes MLX's broken regime)

### Task 4: Four-step FFT for N = n₁·n₂ (each ≤ FFT_BASE_MAX), pow2

**Files:** `shaders/fft.metal`, `crates/polars-metal-kernels/src/fft.rs`, `tests/test_fft.rs`.

Bailey's four-step: view the length-N signal as an n₁×n₂ row-major matrix. (1) FFT each column (length n₁), (2) multiply element (r,c) by `twiddle(r·c, N)`, (3) FFT each row (length n₂), (4) transpose to n₂×n₁ for the correct output order. Steps 1&3 reuse the base kernel (one threadgroup per column/row), so a single four-step level requires n₁,n₂ ≤ FFT_BASE_MAX (=1024) — i.e. it covers `FFT_BASE_MAX < N ≤ FFT_BASE_MAX² = 2^20`. The previously-broken 2^21–2^24 band needs the *recursive* four-step (Task 5); this task establishes the (single-level) four-step machinery and proves it correct up to 2^20.

## Self-review notes (for the executor)

- **Correctness-first ordering:** each phase is independently testable and the differential oracle (DFT for small N, f64 radix-2 for large N, numpy at the Python level) gates every step. Never advance a phase with a failing differential test — that is the whole safety model (CLAUDE.md hand-rolled-kernel discipline).
- **Threadgroup memory (resolved):** `FFT_BASE_MAX=1024` → 2 ping-pong buffers × 1024 × 8 B = 16 KB, safely under the 32 KB threadgroup limit (4096 would be 64 KB — too big). Consequence baked into the plan: a single four-step level covers `N ≤ FFT_BASE_MAX² = 2^20` (Task 4); `N > 2^20` (incl. the broken 2^21–2^24 band) is handled by recursive four-step (Task 5). If you raise `FFT_BASE_MAX`, keep it ≤ 2048 (32 KB exactly — risky) and update both `.metal` and `.rs` + the per-task size bands consistently.
- **`--test-threads=1`** on all `cargo test` ([[polars-metal-test-threading-gotcha]]).
- **Readback:** confirm the real `MetalBuffer` host-readback method name in Task 1 (mirror how `tests/test_rolling.rs` reads output); add `to_f32_vec` if absent.
- **Stress caution:** do NOT crank `PROPTEST_CASES`/giant sizes in a tight loop — Metal accumulates GPU pressure ([[m3-conformance-deferrals]] dispatch-fault note).
