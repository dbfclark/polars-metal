# M5 Rolling Implementation Plan (cumsum-diff via MLX)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transparently accelerate native `pl.col(x).rolling_mean/sum/std/var(window)` under `engine="metal"` via the cumsum-diff identity, fully fused on MLX, matching Polars CPU byte-for-byte (or falling back to CPU).

**Architecture:** A pre-optimization pass (at the `collect` monkey-patch, reading `lf.serialize`) rewrites a handleable `rolling_*` into a **CPU-exact** cumsum-diff expression that the existing M4 fusion engine runs, plus **two new engine ops**: `Shift(w)` (forward shift) and `RowIndex` (`int_range(len)`, an iota). The rewrite is:

```
c = col.cum_sum()
body = c - c.shift(w)                                   # Polars shift null-fills → null for i<w
expr = when(int_range(len) == w-1).then(c).otherwise(body)   # fixes the off-by-one at i=w-1
# mean: expr / w ;  var/std: a second prefix over x² (see Task 10)
```

This expression is **exactly equal to native rolling on CPU** (so any engine fallback is correct — no half-rewritten-expr hole). The engine routes it (`cum_sum`, `shift`, `sub`, `int_range`, `eq`, `Ternary`, `div`); the engine's `Shift` may zero-fill (head values are nulled structurally), and the dispatch applies the **structural first-`w−1` null mask** at fold-back — which matches the expression's intrinsic CPU nulls. `var`/`std` use a second prefix sum over `x²`. Null inputs / non-default options / non-F32 fall back to CPU.

**Tech Stack:** Rust 2021 (`polars-metal-core` fusion engine, `polars-metal-mlx-sys` FFI via `cxx`), MLX (`pad` + slice for shift), PyO3 (`PyFusionScope`), Python (`polars_metal` walker/analyzer/dispatch + pre-opt rewrite), `pytest` + `proptest`.

**Spec:** [`docs/superpowers/specs/2026-06-02-m5-design.md`](../specs/2026-06-02-m5-design.md). Binding. This plan implements **only the rolling sub-project**; `dt` is a separate plan.

```cpp
std::shared_ptr<MlxArray> mlx_shift(const std::shared_ptr<MlxArray>& a, int64_t shift) {
```

```cpp
}
```

```python
# _rolling_rewrite.py
import polars as pl
def _rolling_sum_expr(col: str, w: int) -> pl.Expr:
    c = pl.col(col).cum_sum()
    body = c - c.shift(w)   # Polars shift null-fills → null for i<w; correct for i>=w
    # off-by-one fix: the first full window ends at i=w-1 (value c[w-1]); body is null there.
    return pl.when(pl.int_range(pl.len()) == w - 1).then(c).otherwise(body)

def cumsum_diff_expr(op: str, col: str, w: int) -> pl.Expr:
    s1 = _rolling_sum_expr(col, w)
    if op == "sum":   return s1
    if op == "mean":  return s1 / w
    # var/std handled in Task 10
    raise ValueError(op)
```

> This is **CPU-exact**: Polars `.shift(w)` null-fills, so `c - c.shift(w)` is null for the first `w` rows; the `when(idx == w-1).then(c)` branch restores the first full window. No `fill_null`, no null-producing `Ternary`. # rolling_var = (S2 - S1^2/w) / (w-1), where S1=rolling_sum(x), S2=rolling_sum(x^2)
```python
def _rolling_sum_of(expr: pl.Expr, w: int) -> pl.Expr:
    c = expr.cum_sum()
    body = c - c.shift(w)
    return pl.when(pl.int_range(pl.len()) == w - 1).then(c).otherwise(body)

s1 = _rolling_sum_of(pl.col(col), w)
s2 = _rolling_sum_of(pl.col(col) ** 2, w)
var = (s2 - s1 ** 2 / w) / (w - 1)
return var if op == "var" else var.sqrt()
```
